1 # Setting up ParlAI for a custom chat-service
2 This is an instruction manual to be used as reference on how to configure ParlAI for an arbitrary chat service.
3 
4 ## Message Format
5 To maintain consistency we are trying to enforce a deterministic message format throughout this task. If a particular chat service doesn't adhere to this format, one must use ```restructure_message()``` to adapt the messages to this format before the messages are used. The message format is defined below:
6 ```config
7 {
8  mid: # ID of this message
9  recipient: {
10  id: #id of message recipient
11  }
12  sender: {
13  id: #id of message sender
14  }
15  text: # text of the message
16  attachment: # attachment of the message
17 }
18 ```
19 ### Additional Message fields
20 These define a non-exhaustive list of keys that one could use in the message dict for ease-of-use
21 - messaging_type: whether the message is a text message or an image upload [RESPONSE, UPDATE]
22 - quickreplies: Auto-suggested replies
23 - persona_id: id of the persona that is interacting
24 - name: Display name of the user
25 - profile_picture_url: URL to the profile picture of the user

28 ## Config
29 Below is the standard config format and hierarchy to be followed across all chat services:
30 
31 ```config
32 - tasks: # List of available tasks/worlds for a user to enter
33  - <task 1 name>
34  - onboard_world: # World in which user is first send upon selecting the task. Can collect necessary data from user in this world, as well as provide instructions etc.
35  - task_world: # Actual task world
36  - timeout: # Agent message timeout - how long to wait for the agent to send a message before assuming they have disconnected.
37  - agents_required: # Number of agents required to run the task world, e.g. 2 for a two player game, 1 for a one-player experience, etc.
38  - <task 2 name>
39  - onboard_world:
40  - task_world:
41  - timeout:
42  - agents_required:
43 - task_name: # name of the overall task
44 - world_module: # module in which all of the worlds exist (relative path i.e. `parlai.chat_service....`
45 - overworld: # Name of the overworld; where the agent is first sent upon messaging the service
46 - max_workers: # Maximum number of workers that can be in task worlds at any given moment
47 - opt: # Additional model opts go here. Below are example opts that one could normally pass to parlai
48  - password: # Password for messaging service, if this is wanted
49  - debug: # whether to set debug mode
50  - model: # Name of model, if you want to load a model
51  - model_file: # path to model file, if you want to load a model
52  - override:
53  - model: # overrides for model
54 - additional_args: # Additional chat service specific args go here
55  - service_reference_id: 1 # Facebook Page id (if Messenger, else don't include this field)
56  - *any other args needed by <chat_service>*
57 ```
58 
59 As one can notice, most of the format is the same as how it already exists for messenger with the exception of having ```additional_args:``` as a field in our config. This has been introduced to provide flexibility of parsing any additional arguments a chat service may need whilst preserving the previously existing necessary args. Note however that ```page_id``` has been shifted to this section to maintain coherence.
